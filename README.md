# CPSC 481 — Quantization Benchmark

A benchmark study measuring how **quantization** affects the accuracy, speed, and
output quality of a locally-hosted large language model. The same DeepSeek model is
run at five different quantization levels through [Ollama](https://ollama.com/) and
evaluated against three standard benchmarks.

## What is quantization?

Quantization shrinks a model by storing its weights at lower numerical precision
(e.g. 16-bit floats → 2-bit integers). This dramatically reduces file size and memory
use — and usually speeds up inference — at the cost of some accuracy. This project
measures exactly where that tradeoff lands.

## Models tested

All five are the same DeepSeek model quantized to different levels, registered in
Ollama under these names:

| Ollama name     | Quantization | File size |
|-----------------|--------------|-----------|
| `deepseek-f16`  | F16 (full)   | 16.07 GB  |
| `deepseek-q8`   | Q8_0         | 8.54 GB   |
| `deepseek-q6k`  | Q6_K         | 6.60 GB   |
| `deepseek-q4km` | Q4_K_M       | 4.92 GB   |
| `deepseek-q2k`  | Q2_K         | 3.18 GB   |

## Benchmarks

Each model is evaluated against three [tinyBenchmarks](https://huggingface.co/tinyBenchmarks)
datasets (compact 100-question versions of popular benchmarks):

| Script                    | Dataset                      | Tests                       |
|---------------------------|------------------------------|-----------------------------|
| `benchmark.py`            | tinyAI2_arc (ARC-Challenge)  | Multiple-choice reasoning   |
| `benchmark_gsm8k.py`      | tinyGSM8k                    | Grade-school math word problems |
| `benchmark_truthfulqa.py` | tinyTruthfulQA               | Truthfulness / factuality   |

Each script sends every question to all five models, records the predicted answer,
whether it was correct, and the response time, then writes the results to a CSV.

## Visualization

The `visualize*.py` scripts read the result CSVs and produce graphs comparing the
models on:

- **Accuracy** per quantization level (bar chart)
- **Average response time** per quantization level (bar chart)
- **File size vs. accuracy** tradeoff (scatter plot)

| Script                    | Reads                   |
|---------------------------|-------------------------|
| `visualize.py`            | `results.csv`           |
| `visualize_gsm8k.py`      | `results_gsm8k.csv`     |
| `visualize_truthfulqa.py` | `results_truthfulqa.csv`|

## Qualitative test — `HTMLTEST/`

In addition to the quantitative benchmarks, each model was given the same prompt to
build an HTML calculator. The generated pages in `HTMLTEST/` (one per model) give a
side-by-side look at how code-generation quality degrades as quantization increases.

## Setup

1. **Install [Ollama](https://ollama.com/)** and pull/register the five DeepSeek
   variants under the names listed above (using a `Modelfile` per quantized `.gguf`).
2. **Install Python dependencies:**
   ```bash
   pip install requests pandas datasets matplotlib seaborn
   ```
3. Make sure Ollama is running and serving on `http://localhost:11434`.

## Running

```bash
# Run a benchmark (Ollama must be running)
python benchmark.py              # ARC-Challenge
python benchmark_gsm8k.py        # GSM8K
python benchmark_truthfulqa.py   # TruthfulQA

# Generate graphs from the result CSVs
python visualize.py
python visualize_gsm8k.py
python visualize_truthfulqa.py
```

> **Note:** The scripts read and write CSVs/graphs using hard-coded Windows paths
> (e.g. `C:\CPSC481\results.csv`). Adjust the `csv_path` / save paths in each script
> to match your environment.

## How it works

1. Load 100 questions from the benchmark dataset.
2. For each model, format each question into a prompt and send it to Ollama with
   `temperature = 0` (deterministic output).
3. Parse the model's raw response — stripping DeepSeek `<think>...</think>` reasoning
   tags — to extract the predicted answer.
4. Compare against the correct answer, record correctness and timing, and save to CSV
   after every question.
5. Print per-model accuracy when each model finishes.
